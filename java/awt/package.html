包含用于创建用户界面和绘制图形和图像的所有类。在 AWT 术语中，按钮或滚动条等用户界面对象称为组件。Component 类是所有 AWT 组件的根类。有关所有 AWT 组件共享的属性的详细描述，请参阅 Component。

当用户与组件交互时，某些组件会触发事件。AWTEvent 类及其子类用于表示 AWT 组件可以触发的事件。有关 AWT 事件模型的描述，请参阅 AWTEvent。

容器是一种可以包含组件和其他容器的组件。容器还可以具有控制容器中组件视觉布局的布局管理器。AWT 包含多个布局管理器类和一个用于构建自定义布局管理器的接口。有关更多信息，请参阅 Container 和 LayoutManager。

每个 {@code Component} 对象在最大尺寸和位置上都有限制，因为这些值存储为整数。 此外，平台可能会进一步限制最大尺寸和位置坐标。 确切的最大值取决于平台。 无法通过 Java 代码或本机代码更改这些最大值。 这些限制也对组件布局施加了限制。 如果 Component 对象的边界超过平台限制， 则无法在 Container 对象中正确排列它们。 对象的边界由任何对象的坐标与其在相应轴上的大小的组合定义。
