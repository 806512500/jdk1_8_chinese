拖放是一种在许多图形用户界面系统中发现的直接操作手势，它提供了一种在逻辑上与GUI中的表示元素关联的两个实体之间传输信息的机制。通常由用户使用适当的输入设备进行物理手势驱动，拖放不仅提供了在用户导航GUI中的表示元素时，对任何后续数据传输的可能结果进行连续反馈的机制，还提供了进行任何后续数据协商和传输的设施。

此包定义了在Java中执行拖放操作所需的所有类和接口。它定义了拖放源和放置目标类，以及用于传输被拖动数据的事件。此包还提供了一种在整个拖放操作期间向用户显示视觉反馈的手段。

典型的拖放操作可以分解为以下状态（不一定完全按顺序）：

  • 一个 DragSource 与GUI中的某个表示元素（Component）关联，以启动某些潜在可Transferable数据的拖放操作。

  • 1个或多个DropTarget与GUI中的表示元素（组件）关联，可能能够消费Transferable数据类型。

  • 从DragSource获取一个DragGestureRecognizer，并将其与一个Component关联，以跟踪和识别用户在Component上的任何拖动启动手势。

  • 用户在Component上进行拖动手势，注册的DragGestureRecognizer检测到该手势，并通知其DragGestureListener。

    注意：虽然此API一致地将拖放操作的刺激描述为用户进行的物理手势，但这并不排除在给定适当的DragSource实现的情况下，程序驱动的拖放操作。此包包含用于识别鼠标设备手势的抽象类MouseDragGestureRecognizer。平台可能提供其他抽象子类以支持其他输入设备或特定的Component类语义。

  • DragGestureListener导致DragSource代表用户启动拖放操作，可能动画化GUI光标和/或渲染操作对象的Image。

  • 当用户手势在具有关联DropTarget的GUI中的Component上导航时，DragSource接收通知以提供“拖动经过”反馈效果，而DropTarget接收通知以提供“拖动下方”反馈效果，这些效果基于支持的操作和涉及的数据类型。

手势本身在GUI层次结构上移动一个逻辑光标，与GUI组件的几何形状相交，可能导致逻辑“拖动”光标进入、穿过并随后离开Component及其关联的DropTarget。

DragSource对象通常通过动画化与逻辑光标关联的GUICursor来向用户显示“拖动经过”反馈。

DropTarget对象通常通过在GUI光标下方的关联GUIComponent中渲染动画来向用户显示“拖动下方”反馈。

反馈效果的确定以及数据传输的最终成功或失败（如果发生的话）参数化如下：

  • 由用户选择并由DragSource和DropTarget支持的传输“操作”：复制、移动或引用（链接）。

  • 由DragSource提供的数据类型集与DropTarget可理解的数据类型集的交集。

  • 当用户终止拖动操作时，通常会导致成功的放置，DragSource和DropTarget都会收到通知，这些通知包括并导致通过Transferable对象进行的类型协商和数据传输。
