{@link java.nio.charset.Charset} 字符和字节之间的命名映射
{@link java.nio.charset.CharsetDecoder} 将字节解码为字符
{@link java.nio.charset.CharsetEncoder}   将字符编码为字节
{@link java.nio.charset.CoderResult}   描述编码器结果
{@link java.nio.charset.CodingErrorAction}   描述检测到编码错误时应采取的措施

一个 字符集 是十六位 Unicode 字符序列和字节序列之间的命名映射，如 RFC 2278 中所定义。一个 解码器 是将特定字符集中的字节转换为字符的引擎，而一个 编码器 是将字符转换为字节的引擎。编码器和解码器在字节缓冲区和字符缓冲区上操作。它们统称为 编码器。

{@link java.nio.charset.Charset} 类定义了为给定字符集创建编码器的方法，以及检索与字符集相关联的各种名称的方法。它还定义了静态方法，用于测试特定字符集是否受支持，按名称查找字符集实例，以及构建一个包含当前 Java 虚拟机中所有可用字符集的地图。

大多数用户不会直接使用这些类；相反，他们将使用 {@link java.lang.String} 类中现有的与字符集相关的构造函数和方法，以及现有的 {@link java.io.InputStreamReader} 和 {@link java.io.OutputStreamWriter} 类，这些类的实现已经重新设计，以利用此包中定义的字符集功能。为了允许在这些类的实例构造中指定显式的字符集对象，对 {@link java.io.InputStreamReader} 和 {@link java.io.OutputStreamWriter} 类进行了一些小的修改。

可以通过 {@link java.nio.charset.spi.CharsetProvider} 类在 {@link java.nio.charset.spi} 包中定义的接口来提供对新字符集的支持。
