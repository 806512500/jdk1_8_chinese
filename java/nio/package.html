NIO API 的核心抽象包括：

  • 缓冲区，即数据容器；

  • 字符集 及其 相关的 解码器 和 编码器，
    用于在字节和 Unicode 字符之间进行转换；

  • 各种类型的 通道，表示连接
    到能够执行 I/O 操作的实体；以及

  • 选择器 和 选择键，与
    可选择通道 一起定义了 多路复用、非阻塞
    I/O     设施。

java.nio 包定义了缓冲区类，这些类在整个 NIO API 中使用。字符集 API 在 {@link java.nio.charset} 包中定义，通道和选择器 API 在 {@link java.nio.channels} 包中定义。每个子包都有自己的服务提供者（SPI）子包，其内容可用于扩展平台的默认实现或构建替代实现。

缓冲区 是特定基本类型数据的固定量容器。除了其内容外，缓冲区还有 位置，即下一个要读取或写入的元素的索引，以及 限制，即不应读取或写入的第一个元素的索引。基本 {@link java.nio.Buffer} 类定义了这些属性以及 清除、翻转 和 回绕 的方法，用于 标记 当前位置的方法，以及用于 重置 位置到前一个标记的方法。

每个非布尔基本类型都有一个缓冲区类。每个类定义了一组用于从缓冲区中读取和写入数据的 获取 和 放置 方法，用于 压缩、复制 和 切片 缓冲区的方法，以及用于 分配 新缓冲区和将现有数组 包装 到缓冲区中的静态方法。

字节缓冲区的独特之处在于它们可以用作 I/O 操作的源和目标。它们还支持其他缓冲区类中未提供的几个特性：

  • 字节缓冲区可以被分配为 直接 缓冲区，在这种情况下，Java 虚拟机将尽最大努力直接在其上执行本机 I/O 操作。

  • 字节缓冲区可以通过 {@link java.nio.channels.FileChannel#map 映射} 文件的一个区域直接到内存中创建，在这种情况下，{@link java.nio.MappedByteBuffer} 类中定义的几个额外的文件相关操作可用。

  • 字节缓冲区可以按大端或小端 字节顺序 访问其内容作为 二进制数据 的异构或同构序列。
